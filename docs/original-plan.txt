[Historical snapshot — original conceptual 6-phase plan, since implemented and superseded by the actual built Shopify pipeline; see marketplaces/shopify/README.md for what was actually built and geo-seo-strategy.md for the additional phases added beyond this original scope.]

Phase 0 — Audit (read-only). Write a script that paginates all products via the products query and pulls id, title, productType, category, seo {title, description}, descriptionHtml, tags, vendor plus the relevant metafields. Dump to a CSV/spreadsheet and compute a gap report: which products are missing a meta description, missing a category, have a too-short or duplicated description, etc. This becomes your worklist and your before/after baseline.
Phase 1 — Define the rules per field. Decide the format for each field before generating anything: meta description length and structure (roughly 140–160 chars, lead with the concrete benefit/attribute, include the key search term naturally), a mapping from your internal product types to Shopify taxonomy category IDs, and a title convention (brand + key attributes like material/size/color, which is exactly what Merchant Center rewards). Pull the taxonomy IDs from Shopify's published Standard Product Taxonomy.
Phase 2 — Generate the content. This is the main decision point (see below). Whether AI-drafted or hand-written, the output is a structured file (CSV/JSON) keyed by product ID with the new field values — reviewable before anything gets written.
Phase 3 — Write the update script. Loop the approved file, issue productUpdate mutations. Build in: idempotency (skip if already correct), userErrors checking on every response, rate-limit handling (the GraphQL API is cost-based with a leaky bucket — back off on THROTTLED), and a dry-run mode that logs intended changes without sending. Given the catalog could be large, consider the bulk operations API for the read in Phase 0 and possibly productSet for writes.
Phase 4 — Validate downstream. Use Google's Rich Results / schema validation on a few storefront URLs to confirm the Product markup improved, check the Google & YouTube channel that categories mapped correctly, and watch Merchant Center for diagnostics clearing. Click potential and the Search Console report both lag, so this phase is partly "wait and watch."
Phase 5 — Monitor & iterate. Re-pull click potential (via Merchant API) and the Merchant listings report periodically; feed the low performers back into Phase 2.
