## Group C — on-page JSON-LD enhancement (priceValidUntil + per-variant description + shipping/returns)
**Doing this LAST** (client decision). Owner: Claude writes snippet → client/dev pastes.

ROOT CAUSE (confirmed 2026-06-04 from live HTML): theme is **Dawn 15.4.1**; the Product
JSON-LD is generated by Shopify's built-in **`{{ product | structured_data }}`** filter,
which emits a ProductGroup + hasVariant but OMITS description-per-variant,
priceValidUntil, hasMerchantReturnPolicy, shippingDetails. The filter is NOT
configurable, so it must be REPLACED with a hand-built block. Google validates the
description + priceValidUntil at the **variant** level (hasVariant.offers), which is why
the GSC "description"(146) and "priceValidUntil"(36) warnings persist despite the
group-level description being present.

Snippet written: `shopify/theme/product-structured-data.liquid` — mirrors Dawn's exact
ProductGroup/hasVariant/offers structure and adds, per variant+offer: description,
priceValidUntil, itemCondition, hasMerchantReturnPolicy (US/30-day/customer-paid, matches
MC), shippingDetails (US; **rate is a $0.00 placeholder — set to real US rate or remove
that block so it doesn't mismatch Merchant Center**).

Install (Dawn): in `sections/main-product.liquid` replace the
`<script type="application/ld+json">{{ product | structured_data }}</script>` block with
`{% render 'product-structured-data', product: product %}`; save the snippet as
`snippets/product-structured-data.liquid`; test in Google Rich Results Test. Claude verifies after.

## #3 "Crawl fix" — INVESTIGATED 2026-06-04: NOT actionable / already healthy
Originally framed as "loosen Cloudflare rules." That does NOT apply — the store has no own Cloudflare. DNS: asroutdoor.com A=23.227.38.32 (Shopify IP), nameservers domaincontrol.com (GoDaddy). The `server: cloudflare`/`cf-ray` headers are **Shopify's own Cloudflare** (uncontrollable by merchant). robots.txt is Shopify's healthy default: Googlebot/Storebot-Google NOT crawl-delayed or blocked, product pages crawlable, sitemap listed; crawl-delays apply only to AhrefsBot/MJ12bot/Pinterest (Nutch blocked); AI crawlers (GPTBot/ClaudeBot/PerplexityBot/Google-Extended) fall under `*` = allowed. So the "product page unavailable" spike was a transient Shopify-side throttle of a crawl burst, self-resolving (count declining). **No action: do NOT edit robots.txt; if it ever spikes hard and doesn't recover, open a Shopify Support ticket. Can click "Validate Fix" in GSC to nudge re-crawl.**
